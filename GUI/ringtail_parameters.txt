 -i DATABASE, --input_db DATABASE
                        specify a database file to perform actions with (DONE)
  -s STRING, --bookmark_name STRING
                        Specify name for db view of passing results to create (write mode) or export from (read mode) (DONE)
  -m [dlg] or [vina], --mode [dlg] or [vina]
                        specify AutoDock program used to generate results. Available options are "DLG" and "Vina". Vina mode will automatically change --pattern to *.pdbqt (DONE?)
  -v, --verbose         Print results passing filtering criteria to STDOUT. NOTE: runtime may be slower option used. (DONE)
  -d, --debug           Print additional error information to STDOUT. (DONE)
  -f FILENAME.[DLG/PDBQT][.gz] [FILENAME.[DLG/PDBQT][.gz] ...], --file FILENAME.[DLG/PDBQT][.gz] [FILENAME.[DLG/PDBQT][.gz] ...]
                        ligand docking output file to save. Compressed (.gz) files allowed. Only 1 receptor allowed.
  -fp DIRNAME [DIRNAME ...], --file_path DIRNAME [DIRNAME ...]
                        directory(s) containing docking output files to save. Compressed (.gz) files allowed
  -fl FILENAME [FILENAME ...], --file_list FILENAME [FILENAME ...]
                        file(s) containing the list of docking output files to save; relative or absolute paths are allowed. Compressed (.gz) files allowed
  -p PATTERN, --pattern PATTERN
                        specify which pattern to use when searching for result files to process [only with "--file_path"]
  -r, --recursive       enable recursive directory scan when --file_path is used
  -a, --append_results  Add new results to an existing database, specified by --input_db
  -dh 'ignore' or 'replace', --duplicate_handling 'ignore' or 'replace'
                        specify how duplicate Results rows should be handled when inserting into database. Options are "ignore" or "replace". Default behavior will allow duplicate entries.
  -sr, --save_receptor  Saves receptor PDBQT to database. Receptor location must be specied with in --file, --file_path directory or --file_list file
  -o [FILE_NAME].DB, --output_db [FILE_NAME].DB
                        Name for output database file
  -ov, --overwrite      by default, if a log file exists, it doesn't get overwritten and an error is returned; this option enable overwriting existing log files. Will also overwrite existing database
  -mp INT, --max_poses INT
                        n: Store top pose for top n clusters (DONE)
  -sa, --store_all_poses
                        Store all poses from input files. Overrides --max_poses (DONE)
  -it [FLOAT], --interaction_tolerance [FLOAT]
                        Will add the interactions for poses within some tolerance RMSD range of the top pose in a cluster to that top pose. Can use as flag with default tolerance of 0.8, or give other value as desired. Only compatible with ADGPU mode (DONE)
  -ai, --add_interactions
                        Find interactions between ligand poses and receptor and save to database. Requires receptor PDBQT to be given with input files (all modes) and --receptor_file to be specified with Vina mode. SIGNIFICANTLY INCREASES DATBASE WRITE TIME.
  -ic [HB CUTOFF],[VDW CUTOFF], --interaction_cutoffs [HB CUTOFF],[VDW CUTOFF]
                        Use with --add_interactions, specify distance cutoffs for measuring interactions between ligand and receptor in angstroms. Give as string, separating cutoffs for hydrogen bonds and VDW with comma (in that order). E.g. '-ic 3.7,4.0' will set the cutoff for hydrogen bonds to 3.7 angstroms and for VDW to 4.0. These are the default cutoffs. (DONE)
  -rf STRING, --receptor_file STRING
                        Use with Vina mode. Give file for receptor PDBQT. 
  -mpr MAX_PROC, --max_proc MAX_PROC
                        Maximum number of processes to create during parallel file parsing. Defaults to number of CPU processors. (DONE)

READ ##################################################################################################################################################################################
  -h, --help            show this help message and exit
  -i DATABASE, --input_db DATABASE
                        specify a database file to perform actions with
  -s STRING, --bookmark_name STRING
                        Specify name for db view of passing results to create or export from
  -m [dlg] or [vina], --mode [dlg] or [vina]
                        specify AutoDock program used to generate results. Available options are "DLG" and "Vina". Vina mode will automatically change --pattern to *.pdbqt
  -v, --verbose         Print results passing filtering criteria to STDOUT. NOTE: runtime may be slower option used.
  -d, --debug           Print additional error information to STDOUT.

Output options:
  -l [FILE_NAME].TXT, --log [FILE_NAME].TXT
                        by default, results are saved in "output_log.txt"; if this option is used, ligands and requested info passing the filters will be written to specified file
  -of FIELD1,FIELD2,..., --outfields FIELD1,FIELD2,...
                        defines which fields are used when reporting the results (to stdout and to the log file); fields are specified as comma-separated values, e.g. "--outfields=e,le,hb"; by default, energies_binding (energy) and ligand name are reported; ligand always reported in first column available fields are: "e" (energies_binding), "le" (ligand efficiency),
                        "delta" (delta energy from best pose), "ref_rmsd" (RMSD to reference pose), "e_inter" (intermolecular energy), "e_vdw" (van der waals energy), "e_elec" (electrostatic energy), "e_intra" (intermolecular energy), "n_interact" (number of interactions), "ligand_smile" , "rank" (rank of ligand pose), "run" (run number for ligand pose), "hb"
                        (hydrogen bonds), "receptor" (receptor name); Fields are printed in the order in which they are provided. Ligand name will always be returned and should not be specified
  -ord STRING, --order_results STRING
                        Stipulates how to order the results when written to the log file. By default will be ordered by order results were added to the database. ONLY TAKES ONE OPTION.available fields are: "e" (energies_binding), "le" (ligand efficiency), "delta" (delta energy from best pose), "ref_rmsd" (RMSD to reference pose), "e_inter" (intermolecular energy),
                        "e_vdw" (van der waals energy), "e_elec" (electrostatic energy), "e_intra" (intermolecular energy), "n_interact" (number of interactions), "rank" (rank of ligand pose), "run" (run number for ligand pose), "hb" (hydrogen bonds);
  -oap, --output_all_poses
                        By default, will output only top-scoring pose passing filters per ligand. This flag will cause each pose passing the filters to be logged.
  -xs BOOKMARK_NAME, --export_bookmark_csv BOOKMARK_NAME
                        Create csv of the bookmark given with bookmark_name. Output as <bookmark_name>.csv. Can also export full database tables
  -xq [VALID SQL QUERY], --export_query_csv [VALID SQL QUERY]
                        Create csv of the requested SQL query. Output as query.csv. MUST BE PRE-FORMATTED IN SQL SYNTAX e.g. SELECT [columns] FROM [table] WHERE [conditions]
  -sdf DIRECTORY_NAME, --export_sdf_path DIRECTORY_NAME
                        specify the path where to save poses of ligands passing the filters (SDF format); if the directory does not exist, it will be created; if it already exist, it will throw an error, unless the --overwrite is used NOTE: the log file will be automatically saved in this path. Ligands will be stored as SDF files in the order specified.
  -xdb, --export_bookmark_db
                        Export a database containing only the results found in the bookmark specified by --bookmark_name. Will save as <input_db>_<bookmark_name>.db
  -nd, --data_from_bookmark
                        Write log of --outfields data for bookmark specified by --bookmark_name. Must use without any filters.
  -fb FILTER_BOOKMARK, --filter_bookmark FILTER_BOOKMARK
                        Perform filtering over specified bookmark.
  -p, --plot            Makes scatterplot of LE vs Best Energy, saves as scatter.png.

Property Filters:
  Specify energy and ligand efficiency filters

  -e FLOAT, --eworst FLOAT
                        specify the worst energy value accepted
  -eb FLOAT, --ebest FLOAT
                        specify the best energy value accepted
  -le FLOAT, --leworst FLOAT
                        specify the worst ligand efficiency value accepted
  -leb FLOAT, --lebest FLOAT
                        specify the best ligand efficiency value accepted
  -pe FLOAT, --energy_percentile FLOAT
                        specify the worst energy percentile accepted. Express as percentage e.g. 1 for top 1 percent.
  -ple FLOAT, --le_percentile FLOAT
                        specify the worst ligand efficiency percentile accepted. Express as percentage e.g. 1 for top 1 percent.

Ligand Filters:
  Specify ligand name filter(s)

  -n STRING [STRING ...], --name STRING [STRING ...]
                        specify ligand name(s). Will combine name filters with OR

Interaction Filters:
  Specify interaction filters, either by count or by specific residue interaction. Residue specifications are described using CHAIN:RES:NUM:ATOM_NAME, and any combination is allowed, e.g.: CHAIN:::, :RES::, ::NUM:, :::ATOM_NAME, :RES:NUM:, etc... Unwanted interactions can be defined by prepending "~" to the residue specification, e.g. "~B:THR:276:". Multiple
  residues can be specified in a single option by separating them with a space (e.g.: --vdw=B:THR:276: B:HIS:226:), or by repeating the interaction options (e.g. --vdw=B:THR:276: --vdw=B:HIS:226: ).

  -vdw [-][CHAIN]:[RES]:[NUM]:[ATOM_NAME], --van_der_waals [-][CHAIN]:[RES]:[NUM]:[ATOM_NAME]
                        define van der Waals interactions with residue
  -hb [-][CHAIN]:[RES]:[NUM]:[ATOM_NAME], --hydrogen_bond [-][CHAIN]:[RES]:[NUM]:[ATOM_NAME]
                        define HB (ligand acceptor or donor) interaction
  -r [-][CHAIN]:[RES]:[NUM]:[ATOM_NAME], --reactive_res [-][CHAIN]:[RES]:[NUM]:[ATOM_NAME]
                        check if ligand reacted with specified residue
  -hc NUMBER, --hb_count NUMBER
                        accept ligands with at least the requested number of HB interactions. If a negative number is provided, then accept ligands with no more than the requested number of interactions
  -ra, --react_any      check if ligand reacted with any residue
  -mm INTEGER, --max_miss INTEGER
                        Will separately log all possible combinations of interaction filters in log file excluding up to max_miss numer of interactions from given set. Cannot be used with --plot or --export_sdf_path.
